⚡ Refactor N+1 seed queries into batch transaction

💡 **What:**
Replaced the `for` loop that executed individual `INSERT` statements with a single batch `sql.transaction()` call using Neon's serverless client in `prisma/run_seed.ts`.

🎯 **Why:**
The previous implementation resulted in an N+1 query problem, making a separate HTTP request to the Neon serverless endpoint for every item in `seedData`. By collecting all queries and passing them to `sql.transaction()`, the operation is sent as a single batch HTTP request, dramatically reducing network roundtrips and latency.

📊 **Measured Improvement:**
Since `DATABASE_URL` was not available in the environment to run the seed script directly, no formal benchmark could be gathered. However, assuming an average network latency of 50ms per roundtrip, and ~30 items in the `seedData` array, the original code would take ~1500ms in network overhead alone, whereas the batched code completes the network portion in ~50ms, achieving up to a theoretical 30x improvement in network time and reducing the load on the connection pool.
